# 7天 MVP (最小可行性产品) 上线冲刺计划

## 核心策略：先造“自行车”，再造“法拉利”
要在 1 周内听到第一声响（上线第一条视频），**必须放弃“完整爬取5000条历史数据”和“搭建全量n8n自动化流”的幻想**。

1周内的目标是**跑通核心管线（生成1条真实的视频）**。你需要采用“手动调度 AI”的**MVP（最小可行性产品）策略**：

---

## 📅 Day 1: 确定“爆款”切入点与脚本生成
**目标**：不用写一行代码，通过聊天窗口跟 LLM 协作，把第一个故事的文案敲定。
*   **动作 1（寻找事件）**：去 Wikipedia 搜索这几天（比如你预计发布的那一天）发生的 IT 重大事件。挑出一个最有故事性的（例如：“2001年今天，维基百科正式上线” 或 “1997年今天，苹果收购NeXT让老乔回归”）。**只挑1个！**
*   **动作 2（调教 Prompt）**：打开 Claude 3.5 Sonnet 或 DeepSeek，输入你的背景要求，让其生成一份 250 字左右的口播稿（包含 Hook、高潮、结尾金句）。
*   **动作 3（JSON分镜）**：继续要求大模型：“请将以上口播稿转化为包含时间戳预估、配图提示词（Image Prompt）的 JSON 分镜脚本”。
*   **产出**：一份完美的文案，以及 5 句用来画图的英文 Prompt。

## 📅 Day 2: 视觉素材与音频就位
**目标**：准备好供视频渲染的源材料（图片 + 音频）。
*   **动作 1（获取历史老图）**：去 Google Images 或 Wikimedia Commons 搜索该事件的主角/产品高清老照片（下载 2-3 张真实图片）。
*   **动作 2（AI 补图）**：使用 Midjourney 或 fal.ai 上的 Flux，把昨天的 Image Prompt 丢进去，生成 2-3 张补位的氛围大图（如“赛博朋克风的服务器机房”）。
*   **动作 3（TTS 配音）**：选择一个好用的 TTS 平台（如果不想写代码调用 ChatTTS，可以直接去类似十一言 ElevenLabs 或国内的魔音工坊充值最便宜的套餐，或者使用免费的开源 UI 界面），把文案贴进去，导出为 `.wav` 或 `.mp3` 音频文件。
*   **产出**：一个名叫 `assets` 的文件夹（内含 1段音频 + 5张图片）。

## 📅 Day 3-5: Remotion 模板死磕 (本周最重头戏)
**目标**：不要用剪映！强迫自己用前端代码把这个视频拼出来。这是你核心技术壁垒的起点。
*   **动作 1（初始化环境）**：在你的电脑上运行 `npx create-video@latest`，初始化一个 Remotion 空白 React 项目。
*   **动作 2（挂载素材）**：把 `assets` 文件夹放进 Remotion 项目里，写 `<Audio />` 组件把声音挂上。
*   **动作 3（打字机字幕）**：利用 Remotion 的帧数（fps）计算机制，写一个 React 组建，让字随着音频同步出现。（初期如果搞不定精确字级别时间戳，可以用粗略的按句弹出的组件替代）。
*   **动作 4（镜头缓动）**：写一套全局 CSS 动画（或者用 Remotion 社区的 `ken-burns` 库），让那 5 张图片缓慢放大（Zoom in）和切换，告别死板的幻灯片。
*   **产出**：在浏览器里按空格键，能丝滑预览这条包含声音、字幕和动图的 60 秒视频。

## 📅 Day 6: 渲染导出与人工复查
**目标**：从代码变为物理视频文件，并进行感官质检。
*   **动作 1（渲染成片）**：在命令行输入 `npx remotion render`，等待 FFmpeg 在你的电脑上把 React 代码压制成 1080P 高清或者 4K 的 `.mp4` 文件。
*   **动作 2（质检与微调）**：在手机上播放（必须在手机上播，感受短视频平台的竖屏体感），检查字体是否太大/太小、BGM是否盖过了人声、图片切换是否太快。
*   **动作 3（调整重渲）**：回到代码里改几个参数（改 CSS 比如 `font-size: 80px`），再次按一次 `render` 键。体会代码化渲染的“零废话、零拖拉机操作”的快感。
*   **产出**：一条随时可以发朋友圈的 Final.mp4。

## 📅 Day 7: 账号初始化与多端发布 (0号里程碑)
**目标**：跨出心理门槛，让作品面对市场。
*   **动作 1（账号装修）**：在抖音、小红书、视频号、B站（或YouTube Shorts）注册并装修账号，统一头像、简介（例如：“极客的日历：每天60秒，重温改变世界的IT事件”）。
*   **动作 2（AI 辅助文案）**：把第一天的视频脚本丢给 LLM：“我要发抖音和小红书，请帮我生成配套的标题和爆款 Hashtags（比如 #科技 #IT历史 #程序员）”。
*   **动作 3（手动全网分发）**：手动（暂时不要写 RPA 脚本）把视频上传到各大平台，点击发布。
*   **产出**：全网统一的第一天内容发布成功！

---

## 🚀 为什么要在第 1 周这样做？ (MVP 的意义)
不要一上来就写爬虫去抓 5000 条数据！
1周做完这 **1条** MVP 视频后，你将获得巨大的心理满足感。更重要的是，你验证了**“JSON 渲染模板（Remotion 管线）”**这条最难的路是完全走得通的。

从第二周开始，你的任务才是：**将 Day 1 到 Day 3 的人工动作，一步步写成 Python / n8n 脚本，把这 5000 条数据灌入这条已经被你跑通的“微型工厂”中。**
